# Profile Documentation

*Flexiple Operations Knowledge Base | Category B: Process SOPs*

**Owner:** Hrishikesh | **Last Updated:** 2026-03-12

---

## Overview

A well-documented candidate profile is what the client uses to decide whether to interview. This is Flexiple's "product" - it must be polished, accurate, and actionable.

---

## Must-Haves

- [ ] Candidate's **name**, **current role/title**, and **years of experience**
- [ ] **Tech stack** - clearly separated into primary (daily use) and secondary (familiar with)
- [ ] **Relevant project highlights** - 2-3 bullet points per project, focused on impact
- [ ] **Availability** - start date, notice period, timezone
- [ ] **Rate/salary expectation** - confirmed with the candidate
- [ ] **Flexiple interview notes** - summary of the internal vetting call

## Good-to-Haves

- Candidate's preferred engagement type (contractor vs FTE)
- Communication style assessment (from your call)
- Cultural fit notes (work style, team preferences)
- Portfolio/GitHub links where relevant

## Great-to-Haves

- Quantified impact in project highlights (e.g., "reduced API latency by 40%")
- Client-specific customisation - highlight skills most relevant to *this* client
- Comparison notes if sending multiple profiles ("Candidate A is stronger in X, B in Y")

---

## Do's

- **Do** write in third person, professional tone
- **Do** focus on outcomes and impact, not just responsibilities
- **Do** verify all details with the candidate before sharing with the client
- **Do** proofread - typos in a profile reflect poorly on Flexiple

## Don'ts

- **Don't** copy-paste the candidate's resume - rewrite in Flexiple's format
- **Don't** exaggerate skills or experience
- **Don't** include personal details (age, marital status, religion, etc.)
- **Don't** share a profile without completing the Flexiple interview first

---

## Good Example

> **Priya S. - Senior Full-Stack Developer (6 years)**
>
> **Primary Stack:** React, TypeScript, Node.js, PostgreSQL, AWS
> **Secondary:** Python, Redis, Docker, CI/CD
>
> **Project Highlights:**
> - Led the frontend rebuild of a fintech dashboard serving 50K+ daily users, reducing page load time by 60%
> - Designed and implemented a microservices architecture handling 10M+ API calls/month
> - Built a real-time notification system using WebSockets and Redis pub/sub
>
> **Availability:** 2 weeks notice - IST - Open to contractor engagement
> **Rate:** [amount]
>
> **Flexiple Notes:** Strong communicator, takes ownership. Experienced in client-facing roles. Recommended for senior IC positions.

## Bad Example

> *"Priya is a React developer with 6 years experience. She has worked on various projects. She knows Node.js and AWS. Available soon."*
